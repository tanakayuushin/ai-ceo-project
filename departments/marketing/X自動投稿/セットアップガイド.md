# X 自動投稿セットアップガイド
## Google Apps Script + X API（完全無料）

**作成**: CEO アレン / 2026-05-01  
**費用**: 完全無料　**月間投稿上限**: 約500件（X API無料枠）

---

## 全体の流れ

```
① X Developerアカウント取得（10〜30分）
    ↓
② APIキー4種類を取得
    ↓
③ Googleスプレッドシートを新規作成
    ↓
④ GASスクリプトを貼り付け・設定
    ↓
⑤ トリガーを設定して完了
```

---

## STEP 1：X Developerアカウントを取得する

1. [developer.twitter.com](https://developer.twitter.com) にアクセス
2. 右上「Sign up」→ Xアカウントでログイン
3. 「Free」プランを選択（$0/月）
4. 用途の入力欄に英語で以下を記入：

```
I want to use the X API to automatically post content to my own account.
The posts will be informational tweets about AI tools and business efficiency
for small business owners in Japan. No data collection or analysis of other users.
```

5. 承認メールが届いたら完了（通常即時〜数時間）

---

## STEP 2：APIキーを取得する

1. Developer Portal → 「Projects & Apps」→ 「New App」
2. アプリ名を入力（例：`my_auto_post`）
3. 「Keys and Tokens」タブをクリック
4. 以下の4つをコピーしてメモ帳に保存：

| 項目 | 場所 |
|------|------|
| Consumer Key | API Key & Secret |
| Consumer Secret | API Key & Secret |
| Access Token | Authentication Tokens |
| Access Token Secret | Authentication Tokens |

> ⚠️ **重要**: 「App permissions」で **「Read and Write」** に変更すること。  
> デフォルトは「Read」のみのため、投稿できません。

---

## STEP 3：Googleスプレッドシートを作成する

1. [Google Drive](https://drive.google.com) で「新規」→「スプレッドシート」
2. ファイル名を「X自動投稿管理」にする
3. 上部メニュー「拡張機能」→「Apps Script」をクリック
4. 開いたエディタの左側「コード.gs」をクリック
5. デフォルトのコードを**全部削除**する

---

## STEP 4：スクリプトを貼り付ける

1. `auto_post_x.gs` の内容を**全部コピー**してエディタに貼り付ける
2. 「プロジェクトを保存」（Ctrl+S）

### APIキーを登録する

`setup()` 関数内の4箇所を自分のキーに書き換える：

```javascript
props.setProperties({
  'X_CONSUMER_KEY':        'ここに Consumer Key',
  'X_CONSUMER_SECRET':     'ここに Consumer Secret',
  'X_ACCESS_TOKEN':        'ここに Access Token',
  'X_ACCESS_TOKEN_SECRET': 'ここに Access Token Secret',
});
```

書き換えたら、関数名のドロップダウンで「setup」を選んで▶実行ボタンを押す。

> 初回実行時に「Googleアカウントへのアクセスを許可」が求められる → 「許可」をクリック

---

## STEP 5：スプレッドシートのひな形を作る

1. ドロップダウンで「createSheet」を選んで▶実行
2. スプレッドシートに「投稿予定」シートが自動作成される

### スプレッドシートの入力方法

| A列（投稿予定日時） | B列（投稿内容） | C列（ステータス） |
|-------------------|--------------|-----------------|
| 2026/05/01 09:00 | ツイートの内容（140字以内） | 空白のままでOK |
| 2026/05/01 12:00 | 次のツイート内容 | |

- A列は日時形式で入力（スプレッドシートが自動認識）
- B列に実際の投稿内容を書く
- C列は自動で「投稿済」になる（触らない）

---

## STEP 6：トリガーを設定する（自動化の核心）

1. ドロップダウンで「setHourlyTrigger」を選んで▶実行
2. 「毎時トリガーを設定しました」とログが出たら完了

これで **毎時間0分に自動的に `checkAndPost` が動き**、  
スプレッドシートの予定時刻を確認して投稿します。

---

## STEP 7：テスト投稿で動作確認

1. ドロップダウンで「testPost」を選んで▶実行
2. Xアカウントに「動作確認」ツイートが投稿されれば成功
3. 確認できたらそのツイートは手動で削除する

---

## 運用フロー（週次作業）

```
月曜日：1週間分の投稿内容をスプレッドシートに入力（15〜30分）
  ↓
あとはGASが自動投稿
  ↓
金曜日：反応が良かった投稿をチェック → 翌週のネタに活かす
```

## トラブルシューティング

| 症状 | 原因 | 対処 |
|------|------|------|
| 「APIキーが未設定」と出る | setup()未実行 | setup()を実行 |
| 投稿されない | 時刻のズレ | A列が日時形式か確認。±10分以内に合わせる |
| 「401 Unauthorized」エラー | 権限が「Read」のまま | DeveloperポータルでRead&Writeに変更し、キーを再発行 |
| 「429 Too Many Requests」 | 月500件の無料枠超過 | 翌月まで待つ or Basic($100/月)に移行 |
