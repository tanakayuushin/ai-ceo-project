# ококоナラ 公開依頼への提案文テンプレート

**作成**: CEO アレン / 2026-04-30  
**根拠**: 実績出品者の提案パターン分析・高採用率テンプレート研究

---

## 使い方

1. 公開依頼のページを読む
2. 依頼内容に最も近いテンプレートを選ぶ
3. **【 】の部分を必ず書き換えてから**送信する
4. 文字数：300〜400文字を目安にする

> ⚠️ コピペそのままは絶対NG。「この人は読んでくれた」と感じさせる一言を必ず冒頭に入れること。

---

## 提案文の必須構成

```
① 読んだことを示す一言（依頼内容への共感・具体的な言及）
② 自分の強みと解決できること
③ モニター価格の明示と理由（信頼感につながる）
④ 行動を促すひと言（「まずはご相談ください」）
```

## テンプレートA：AI活用相談（サービス1）

### 依頼例：「ChatGPTを使い始めたが活用できていない」「AIをどこに使えばいいかわからない」

```
【依頼者の状況の一言】のご状況、よく理解できます。

私はAI活用のサポートを専門としており、同じお悩みを持つ方に、
「何から手をつければいいか」を一緒に整理することが得意です。

チャットでヒアリングしながら、【依頼内容に関連する具体的なアドバイス例】
といった形で、実際にすぐ使えるアドバイスをお伝えします。

現在、実績作りのためのモニター価格（2,000円）でご提供中です。
低リスクで試していただけます。

まずはお気軽にご相談ください。ご返信お待ちしております。
```


## 提案時のチェックリスト

送信前に必ず確認すること：

- [ ] 依頼内容の具体的な部分に触れているか（コピペに見えないか）
- [ ] 「自分がどう解決するか」が明確に書かれているか
- [ ] モニター価格の理由（実績作り中）を伝えているか
- [ ] 300〜400文字に収まっているか
- [ ] 末尾に「まずはお気軽にご相談ください」が入っているか
- [ ] 「！」や絵文字の多用をしていないか（押しつけがましく見える）

---

## 採用率を上げるNG集（実績者分析より）

| NG | 理由 |
|----|------|
| 冒頭が「はじめまして」だけ | 読んでいない感が出る |
| 自己紹介が長すぎる | 依頼者の課題より自分の話が多い |
| 「どんな依頼でも対応します」 | 専門性がなく見える |
| 料金だけ強調する | 安さで選ばれる出品者になってしまう |
| 500文字超え | 読まれない |
| 「★5をいただけると嬉しいです」 | 規約違反・NG |

## 公開依頼の探し方

1. ококоナラ上部メニュー「依頼を探す」→「公開依頼」
2. カテゴリ：「IT・テクノロジー」→「AI導入・生成AI活用相談」を優先
3. キーワード：「ChatGPT」「AI活用」「業務効率化」「生成AI」
4. 投稿が新しい順（24時間以内）を優先して提案する
5. **1日3〜5件を目安に毎日提案を続ける**（量より継続が大事）
